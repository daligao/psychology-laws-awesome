# Contributing to Awesome Psychology Laws

Thank you for interest in contributing! This guide explains how to add to this collection.

## Before You Contribute

- [ ] Read the main [README.md](README.md)
- [ ] Understand the 40 existing laws and their categories
- [ ] Review one existing entry for formatting
- [ ] Check if your addition is truly psychology-related (not just motivation/self-help)

## Types of Contributions

### 1. Add a New Psychology Law (High Priority)

If you know a psychology law/bias that should be in the list:

1. **Verify it's scientifically grounded** — Must be from psychology research, not pop psychology
2. **Write the entry** with:
   - Law name
   - One-sentence definition
   - Link to detailed article (or propose one)
   - Real-world example
3. **Place it in correct category**:
   - Cognitive Biases
   - Decision-Making
   - Social & Behavioral
   - Logic & Reasoning

### 2. Improve Existing Entries

- Add better examples
- Fix broken links (report to ordinarymantrying.com)
- Clarify definitions
- Add missing citations

### 3. Add Interactive Content

- Suggest new quiz scenarios for Mind Traps
- Propose new games or learning tools
- Create study guides

### 4. Improve Documentation

- Translate to other languages
- Create learning paths
- Add study guides

## Submission Guidelines

### Quality Standards

**✅ Good contributions:**
- Based on published psychology research
- Clear, concise language
- Specific real-world examples
- Properly linked
- Factually accurate

**❌ Avoid:**
- Pop psychology / unsourced claims
- Self-promotion (avoid "my blog")
- Duplicate laws (check existing list first)
- Vague definitions
- Broken links

### Format Requirements

When adding a law:

```markdown
| # | Law | Description | Link |
|---|---|---|---|
| [N] | [Law Name] | [1-2 sentence definition] | [https://link-to-article] |
```

Example:
```markdown
| 41 | Recency Bias | Overweighting recent information in decisions | [Read More](https://example.com/recency-bias/) |
```

## How to Submit

### For Adding Laws:

1. Fork the repository
2. Add your law to the appropriate category table
3. Verify link is working
4. Submit a Pull Request with:
   - Title: `Add: [Law Name]`
   - Description: Why this law is important + source
   - Confirm it's not a duplicate

### For Fixing Issues:

1. Fork the repository
2. Make your changes
3. Submit a PR with:
   - Title: `Fix: [description]`
   - Description: What was wrong + how you fixed it

### For Large Changes:

Open an Issue first to discuss before working.

## PR Review Process

We'll review your PR for:

1. **Accuracy** — Is the law definition correct?
2. **Relevance** — Is it truly a psychology law?
3. **Quality** — Is the description clear?
4. **Links** — Do they work?
5. **Formatting** — Does it match the style?

Typical review time: 3-7 days.

## Expectations

- Be respectful and constructive
- Respond to feedback on your PR
- Be patient — maintainers review in free time
- One good contribution is better than many mediocre ones
